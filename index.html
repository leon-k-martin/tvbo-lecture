1. Intro to Brain Network Modelling

Intro to Brain Network Modelling

​​​​​​

The opener for the whole course — build it up live, one click at a time: first the brain (the cortical surface), then the wiring (white-matter tracts connecting regions), then the dynamics (activity firing and travelling along those tracts), and finally the network itself (each region as a node). That progression is brain network modelling: anatomy gives the scaffold, the connectome gives the edges, and local dynamics on each node give the behaviour. Everything in the rest of the course hangs on this picture.

Why network models? Same lesion, different outcomes

  • Two patients, similar lesion location and volume
  • Different deficits, different recovery trajectories
  • The locus alone doesn’t predict outcome
  • Function lives in the network: regional damage propagates through the connections that survive

To predict outcome, model the whole network, not a catalogue of regions.

Why dynamics? The brain is never idle

  • At rest, the brain shows structured spontaneous activity, not noise
  • Resting-state networks are reproducible across subjects and species
  • Tasks are perturbations of this ongoing dynamical state
  • Like a tennis player at the baseline: never still, poised for the next service

A model has to reproduce the resting repertoire, not only stimulus responses.

Outputs match what we measure

  • TVB outputs what scanners record: EEG · MEG · SEEG · fMRI BOLD
  • Each modality is a forward model (\(h\)) on the same simulated state
  • Timescales come for free, EEG/SEEG in ms, BOLD in seconds

Direct comparison with data, no translation gap.

Towards Realism — Damping & Load

Now we modify the toy model toward a more realistic oscillator.

What changes?

\[ \dot{v} = -\frac{k}{m}x - \frac{c}{m}v + g_{\mathrm{eff}} \]

  • Add constant load: \(g_{\mathrm{eff}}=0.00015\)

What do we see?

  • Oscillations decay over time
  • Equilibrium shifts away from zero

EEG-Like Activity with the Jansen-Rit-Model

Jansen and Rit (1995) Lumped-parameter cortical column model with three neural populations: pyramidal cells, excitatory interneurons, and inhibitory interneurons. Each population is modeled by a PSP block (2nd-order linear filter) and a sigmoidal potential-to-firing-rate transformation.

Derived Variables \[ v_{pyr} = y_{1} - y_{2} \]

State Equations \[ \dot{y_{0}} = y_{3} \] \[ \dot{y_{3}} = - y_{0}*a^{2} - 2*a*y_{3} + A*a*\operatorname{Sigm}{\left(y_{1} - y_{2} \right)} \] \[ \dot{y_{1}} = y_{4} \] \[ \dot{y_{4}} = - y_{1}*a^{2} - 2*a*y_{4} + A*a*\left(p + C_{2}*\operatorname{Sigm}{\left(C_{1}*y_{0} \right)} + K*c_{intercolumn}\right) \] \[ \dot{y_{2}} = y_{5} \] \[ \dot{y_{5}} = - y_{2}*b^{2} - 2*b*y_{5} + B*C_{4}*b*\operatorname{Sigm}{\left(C_{3}*y_{0} \right)} \]

with \[ \operatorname{Sigm}{\left(v \right)} = \frac{2*e_{0}}{1 + e^{r*\left(v_{0} - v\right)}} \]

Kuramoto (1975) — same model as before, now with coupling enabled.

Mathematical framework

\[dS_i = \left[f_d(S_i, \theta^d, C_i, I_i) \right]dt + g(S_i, \theta^g)\, dW_i\]

State Evolution

  • \(S_i\) - state variables at node \(i\)
  • \(f_d\) - dynamics function with parameters \(\theta^d\)
  • \(C_i\) - coupling input from connected nodes
  • \(I_i\) - external input (stimulation, driving signals)
  • \(g\) - noise diffusion coefficient with parameters \(\theta^g\)
  • \(dW_i\) - Wiener process (stochastic fluctuations)

Coupled Brain Network

\[ C_i = f_c^{\text{post}}\Big(\sum_j A_{ij}\, f_c^{\text{pre}}(S_i, S_j(t-\tau_{ij}), \theta^c), S_i, \theta^c\Big) \]

Coupling

  • \(f_c^{\text{pre}}\) - pre-aggregation transformation
  • \(A_{ij}\) - structural connectivity weight (\(j \to i\))
  • \(\tau_{ij}\) - transmission delay (tract length / conduction speed)
  • \(f_c^{\text{post}}\) - post-aggregation transformation
  • \(\theta^c\) - coupling parameters

The Virtual Brain Ontology (TVB-O)

A brain network model usually lives in a methods section — prose you can read but rarely re-run. TVB-O makes the model itself a FAIR, machine-readable artifact:

  • Common vocabulary — ontology-backed terms with persistent identifiers (w3id.org/tvbo); no more ambiguous symbols across papers
  • Machine-readable metadata — a compact YAML schema captures everything needed to reproduce a simulation
  • Curated database — published models, connectomes and study configurations, ready to load and run
  • Code generation — one specification → executable code for TVB, JAX, Julia, …

The single slide that introduces the ontology the whole course builds on. The problem: a model published in a paper is prose plus symbols — human-readable but not machine-actionable, so reproducing it is slow and error-prone. TVB-O makes the specification itself the shareable object, and four things follow: a common ontology-backed vocabulary (terms resolve to persistent identifiers), a compact machine-readable YAML schema, a curated database of ready-to-run models and connectomes, and code generation to multiple backends (TVB, JAX, Julia). The next slides show what that specification looks like in practice.

Python API

from tvbo import DynamicalSystem, SimulationExperiment
from tvbo.datamodel.schema import Exploration, ExplorationAxis

bsplot.style.use("tvbo")

g = DynamicalSystem(
    state_variables={"x": {
                          "equation": {"rhs": "a*x"},
                          "initial_value": 1.0}},
    parameters={"a": {"value": 1.0}},
)

exp = SimulationExperiment(dynamics=g)

exp.explorations = Exploration(
    name="a_sweep",
    space=ExplorationAxis(
      parameter="a",
      explored_values=[1.0, 0, -1.0]
      ),
)

exp.integration.duration = 4.0
exp.integration.step_size = 0.01

res = exp.run("tvboptim")

In short

A brain network model is local dynamics on a structural connectome, observed through a forward model.

  • Regions carry neural-mass dynamics — the local node model
  • Wired by the connectome — weights and conduction delays
  • A forward model maps activity to BOLD, EEG, MEG

TVB-O isn’t a piece of software — it’s a whole framework that harmonizes how such models are specified, so any of them stays shareable and re-runnable.

Recap of the framework before we turn to applications: a brain network model couples local regional dynamics over the structural connectome — with connection weights and conduction delays — and a forward model turns the simulated activity into measurable signals like BOLD, EEG or MEG. That whole construct is what’s rotating here. TVB-O is not a tool you run; it’s a framework for harmonizing how those models are specified, so any of them can be shared and re-run.

4. Clinical Applications

From mechanism to the clinic

Brain disorders are disorders of brain dynamics. A personalized virtual brain turns a patient’s scans into a working model — to expose a mechanism we cannot measure, and to rehearse a treatment before it reaches the patient.

One idea for the whole chapter: brain disorders are disorders of dynamics — so rebuild each patient’s brain as a network model and let it run.

  • Build — patient scans (MRI · dMRI · EEG/MEG) → a personalized virtual brain
  • Simulate & fit — match the model to that patient’s data
  • Infer — read off a mechanism you can’t measure directly: excitability · E/I balance · effective connectivity
  • Use — predict an outcome, or rehearse a stimulation/therapy in silico before touching the patient

The dashed arrow = the personalized-medicine loop: the clinical question shapes the next measurement. Next slides: where this recipe has actually reached patients, disease by disease.

5. Bifurcation Analysis

How to analyse a brain system

Time series \(x(t)\) at fixed \((a, x_0)\)

Phase plane flow geometry at fixed \(a\)

Bifurcation diagram regimes as \(a\) varies

Three complementary lenses on the same system. The time series is what you’d actually record — but it only shows one parameter setting. The phase plane shows the geometry of the flow at a fixed parameter: where trajectories go, what’s stable. The bifurcation diagram is the summary — it tracks how the equilibria and their stability change as we turn one knob (\(a\)). Goal for this chapter: learn to read that third panel. It’s a map of regimes, and it tells you what behaviours a model can produce before you run a single long simulation.

Linear Systems

\[ \dot x = a\,x \quad\Rightarrow\quad x(t) = x_0\,e^{a t} \]

  • \(x(t)\): State variable
  • \(x_0\): Initial condition
  • \(a\): Parameter

Solution: \[ x(t) = x_0\,e^{a t} \]

Equilibrium: \[ \dot{x}(t) = ax(t) = 0? \] \[ \quad\Rightarrow\quad x_{eq} = 0 \]

Start with the simplest possible system — 1D and linear. Everything about stability is in the sign of \(a\). The solution is an exponential: if \(a<0\) any perturbation decays back to zero, so the equilibrium is stable; if \(a>0\) it grows, so it’s unstable; \(a=0\) is the knife-edge. The equilibrium is always \(x=0\), but its stability flips as \(a\) passes through zero. The second figure makes this concrete: nudge the system off the equilibrium and, when it’s stable, it returns. Neural masses work the same way — we linearise around a fixed point, and the sign of the eigenvalues tells us whether activity settles back or runs away.

2D portraits

Two ways to read the same phase portraits. First the animation: watch the flow reorganise as the parameter sweeps — fixed points appear, move, change stability, and a closed loop (a limit cycle) emerges. Then the static snapshots: the same regimes frozen side by side, left-to-right from a single stable resting state, through the bifurcation, to a sustained oscillation. Takeaway: small smooth parameter changes cause qualitative changes in behaviour — that’s a bifurcation.

Regime-change building blocks

  • Top: equilibria as \(a\) varies.
  • Bottom: local flow \(\dot x=f(x)\) at the dotted parameter slice.

There’s only a small zoo of local bifurcations — the normal forms. Top panel: equilibria and their stability (solid = stable, dashed = unstable) as the parameter varies. Bottom: the 1D flow at the dotted slice. The classics are saddle-node (two fixed points collide and vanish), transcritical (they swap stability), pitchfork (one splits into three), and Hopf (next slide). You don’t have to memorise them — just know that almost every regime change in a neural model is one of these in disguise.

Hopf: birth of an oscillation

\[ \dot z = (a + i\omega)z - |z|^2z, \qquad r_{\mathrm{cycle}} = \sqrt a \]

fixed point loses stability \(\Rightarrow\) stable oscillation appears

This is the important one for brain dynamics, because it’s how rhythms are born. As \(a\) crosses zero, a stable fixed point loses stability and a stable limit cycle appears around it, with radius growing like \(\sqrt a\) — so the oscillation starts small and grows. Mechanistically: the system stops settling to a constant rate and starts oscillating (think alpha or gamma rhythms emerging). One-sentence version: a Hopf bifurcation is where a quiet equilibrium turns into a spontaneous oscillation.

Dynamical Regimes

Tie it back to the time series — the thing experimentalists actually see. Each parameter regime produces a characteristically different trace: a flat/noisy fixed point, a damped transient, or a sustained oscillation. The bifurcation diagram predicts which of these you’ll get, so instead of guessing parameters by trial and error you read them off the map.

Numerical continuation

\[ F(x, a)=0, \qquad (x,a)=(x(s),a(s)) \]

  • follow branches through folds
  • detect folds and Hopf points
  • switch to periodic orbits

How do we compute these diagrams without brute-force simulating millions of settings? Continuation. We solve \(F(x,a)=0\) and follow the solution branch as \(a\) changes — like walking along a ridge. Pseudo-arclength parametrises by arc length \(s\) so we can turn around “folds” where the curve bends back on itself (a plain parameter sweep would just fail there). Along the way we automatically detect folds and Hopf points and can switch onto the periodic-orbit branch. This is what tools like BifurcationKit.jl do.

Each node is coloured by its instantaneous state and placed on the shared bifurcation diagram — so you can literally watch which regime each region is in, and how coupling pushes nodes across bifurcations. The point: bifurcation analysis isn’t just a single-node toy, it organises the dynamics of the full network model.

7. Stimulating the Brain

Why stimulate a brain model?

  • At rest the brain is a dynamical system poised near a regime boundary (Intro, Bifurcation)
  • Tasks, sensory flashes, TMS, and DBS are all perturbations of that ongoing state
  • A stimulus lets us probe the regime — does a nudge decay, or tip the system into a new behaviour?
  • … reproduce evoked responses we record (EEG/MEG/SEEG)
  • … and test interventions in silico before touching a patient

Tie this straight back to chapters 1 and 5. In chapter 1 we said the brain is never idle — resting activity is structured, and tasks are perturbations of that ongoing dynamical state. In chapter 5 we learned to read a model as a map of regimes. Stimulation is where those two ideas meet: we inject an external signal and watch how the system responds. The same machinery covers a visual flash, a TMS pulse, or a DBS train — they differ only in where, when, and how strong. The figure is the chapter-5 “nudge the equilibrium” picture; stimulation just makes that nudge an explicit, time-dependent input we control.

Stimulation is a time-dependent Event

In TVB-O a stimulus is an Event: a symbolic, time-dependent perturbation injected into the dynamics.

field meaning
name identifier — and the model port it drives
event_type "stimulus" for external input
parameters named knobs (onset, width, amplitude, …)
equation.rhs any expression in t and the parameters
nodes / weights where it lands — which nodes, how strongly

Because the signal is symbolic, the same object serializes to YAML, renders to every backend, and is fully FAIR — just like the model itself.

The key idea: a stimulus is not a hardcoded array of numbers, it’s a symbolic equation in time. That’s what makes it portable and reproducible — it round-trips through the ontology like every other part of the experiment. The fields map onto the three questions any stimulus has to answer: what shape (equation + parameters), where (nodes/weights), and into what (the model port its name binds to). Don’t dwell on the table — we’ll see the main fields appear in the next slides.

The recipe: define → inject → run

  1. Define the signal as an Event — a shape in time
  2. Inject it: register the event and add its variable to a state equation’s right-hand side
  3. Run and compare the resting vs. stimulated time series

We start with a single Generic-2d-Oscillator node — the same model whose bifurcation diagram we read in Bifurcation Analysis — then place it in a small network.

Step 0 — A baseline run

from tvbo import Dynamics, SimulationExperiment
from tvbo.datamodel.schema import Event

g2d = Dynamics.from_db("Generic2dOscillator")
exp = SimulationExperiment(dynamics=g2d)

res = exp.run()
res.integration.sel(variable="V").plot()

With its database defaults (\(I = 0\)) the node relaxes onto a stable focus — far from any oscillation. This is our resting state.

Before stimulating, establish the baseline. Loaded straight from the database, the Generic-2d oscillator sits on a stable fixed point: whatever the initial transient, it settles to a constant — the “quiet equilibrium” from the Hopf slide in chapter 5. Everything interesting on the next slides is the departure from this flat line.

Step 1 — Define the stimulus

A rectangular pulse, written as a Piecewise in t:

event = Event(
    name="stim",
    event_type="stimulus",
    parameters={
        "onset":     {"value": 200.0},
        "width":     {"value": 200.0},
        "amplitude": {"value": 5.0},
    },
    equation={"rhs":
        "Piecewise((amplitude,"
        " (t >= onset) & (t < onset + width)),"
        " (0.0, True))"},
)
event.plot()

Any expression in t and the event’s parameters is valid — a Gaussian, a ramp, or a sinusoid works the same way.

The equation is the whole stimulus: a box at amplitude while onset ≤ t < onset+width, zero otherwise. The parameters are named knobs, so onset/width/amplitude become things you can sweep or even infer later (the bridge to chapter 6). Because it’s symbolic, a Gaussian bump or an oscillating drive is a one-line change to the rhs — nothing else in the recipe moves.

Step 2 — Expose a port, bind the event

The model author places a stimulation port stim (default 0) where input physically enters; the event binds to it by name.

from tvbo.datamodel.schema import Parameter

# the model exposes a stimulation port `stim`, defaulting to 0
g2d.parameters["stim"] = Parameter(name="stim", value=0.0)
g2d.state_variables["V"].equation.rhs = (
    "d*tau*((I + stim)*gamma - V**3*f + V**2*e + V*g "
    "+ V*local_coupling + W*alpha + c_glob*gamma)"
)

exp.events["stim"] = event   # event.name == port symbol → bound

The port is authored once, at the right spot in the equation. With no event stim = 0 → the resting model; attaching the event drives it. The equations are never rewritten per experiment, and nothing is injected implicitly. (Jansen-Rit ships exactly such a port — P — see the use-case.)

This is the conceptual heart of the chapter, and the clean design. Rather than mutating the model’s equation per experiment, the model exposes a port — a named input symbol (stim) placed where external drive physically enters, defaulting to zero. The event binds to that port by name. Two payoffs: the model stays invariant (no event → stim=0 → rest, attach event → driven, no equation surgery), and placement is explicit, so there is no hidden additive assumption — the author decides where and how the input enters. This is exactly the P input port the Jansen-Rit model ships with.

Step 3 — Run: rest → stimulus → rest

While the pulse is on, it lifts the effective input \(I + \text{stim} = 5\) above the node’s Hopf bifurcation (\(\approx 3.5\)), into the limit-cycle envelope (right). A limit cycle is born for the pulse’s duration, then activity relaxes back to rest — the Hopf mechanism from Bifurcation Analysis, triggered on demand.

Payoff slide. Left: the trace is flat, oscillates during the grey pulse window, then flattens again. Right: why — the bifurcation diagram of this exact G2D. Rest sits at I=0 on the stable branch (green); the pulse adds 5, landing at I=5 inside the limit-cycle envelope (red), past the Hopf at ~3.5. So stimulation and bifurcation analysis are the same story from two sides: the diagram predicts that a strong-enough pulse triggers oscillation, and the time series shows it happen.

From one node to a network

Localize the stimulus to one node with nodes / weights:

event = Event(
    name="stim", event_type="stimulus",
    parameters={"onset": {"value": 80.0},
                "duration": {"value": 5.0},
                "amplitude": {"value": 5.0}},
    equation={"rhs": "Piecewise((amplitude,"
        " (t>=onset)&(t<onset+duration)),"
        " (0.0, True))"},
    nodes=[0],      # stim at R0
    weights=[1.0],  # full strength
)
net = Network.from_string(motif_yaml)

R0 → R1 (short 30 mm), R0 → R2 (long 200 mm) — delay = length / conduction_speed.

Two Event fields do all the spatial work: nodes picks which nodes the stimulus lands on, weights how strongly. Everything else is identical to the single-node case. The motif is deliberately tiny — one stimulated source R0 feeding a near node (30 mm) and a far node (200 mm). Because the transmission delay is length over conduction speed, the far branch should lag the near one. The overview’s length matrix shows the 30-vs-200 mm asymmetry; the next slide turns that into a timing difference.

Conduction speed shapes propagation

res_fast = stimulate(speed=10)   # fast conduction (10 mm/ms) → near-synchronous
res_slow = stimulate(speed=1)    # slow conduction  (1 mm/ms) → staggered arrival

Same localized pulse, same tracts — only conduction_speed changes. Fast: R1 and R2 respond almost together. Slow: the far node R2 (200 mm) lags the near node R1 (30 mm) by ≈ 170 ms — the network’s geometry written directly into the timing.

The payoff for the network. Identical experiment, one parameter changed. At fast conduction the 30 mm and 200 mm delays are both tiny, so R1 and R2 fire almost together. Drop the speed to 1 mm/ms and the delays become 30 ms and 200 ms — now the far node R2 visibly lags the near node R1 by about 170 ms. Same stimulus, same wiring; the conduction speed alone reshapes the spatiotemporal response. This is the delayed propagation the Jansen-Rit visual-evoked-potential model on the next slide relies on.

Secondary propagation: a 1 → 2 → 3 chain

  • 1 → 2 → 3 · node 3 reached only via node 2 → secondary (poly-synaptic) propagation
  • per hop: later \(t_k \approx t_0 + k\,\tau\) · weaker \(A_k \sim g^{\,k}\ (g<1)\)
  • same weights & coupling as the conduction-speed demo
  • → indirect routes in a full connectome

The concept slide for multi-hop propagation. Left: a minimal 1 → 2 → 3 chain, stimulus on node 1. Right: node 1 fires first, then node 2, then node 3 — a travelling wave down the chain. The key point is secondary propagation: node 3 has no direct edge from node 1, so its response is entirely inherited through node 2. In a real connectome this is how a focal stimulus (or a lesion’s effect) reaches regions it isn’t directly wired to — along indirect, multi-synaptic paths.

Params match the conduction-speed demo exactly (linear coupling a=3, unit weights), so the response attenuates per hop. (An earlier version used a=3→6 and the response grew; that growth is just coupling gain exceeding damping, not phase-matching — verified by sweeping the delay, the amplitude is delay-independent.)
